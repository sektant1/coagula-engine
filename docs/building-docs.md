# Building the Docs

This site is generated by Doxygen from the engine headers + handwritten
Markdown under `docs/`.

## Requirements

- Doxygen ≥ 1.9.
- Optional: Graphviz `dot` for call/include graphs.

If Doxygen is not installed at CMake-configure time, the `engine-docs`
target is simply not created.

## Build

```sh
cmake -S . -B build
cmake --build build --target engine-docs
```

Output:

```
docs/generated/html/index.html
```

Open that file in a browser. No server needed.

## What gets indexed

Configured in `Doxyfile.in` at the repo root:

- `engine/source/` — all engine headers + `.cpp` + `modules.dox`.
- `source/` — executable entrypoint.
- `README.md` — used as the main page.
- `docs/*.md` — handwritten pages (overview, build, layout, starting point, tutorials, recipes, guides, license, this page).

Excluded: `thirdparty/`, `build/`, `bin/`, `docs/generated/`.

## Writing a new page

1. Create `docs/my-page.md`.
2. First line declares the page id and title:

   ```md
   @page my_page My Page Title
   @tableofcontents
   ```

3. Add it to `INPUT` in `Doxyfile.in`.
4. Link it from `docs/overview.md` or another page with `@ref my_page`.
5. Re-run the `engine-docs` target.

## Cross-references

- `@ref coa_render` — link to a `@defgroup` (see `engine/source/modules.dox`).
- `COA::Scene::Load` — bare symbol names auto-link.
- `@ref recipes` / `@ref build` — link to Markdown pages.

## Styling

`docs/custom.css` is applied on top of Doxygen's dark theme. Keep overrides
minimal — broad selectors collide with Doxygen's own layout rules.

## Regenerating after edits

Only the `engine-docs` target needs to run; the executable is not rebuilt:

```sh
cmake --build build --target engine-docs
```
